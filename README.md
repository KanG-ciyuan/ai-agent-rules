# AI Agent 执行规范

一套可复用的 AI Agent 行为准则——用结构化确认约束输出空间，用分阶段门禁防止跳步和脑补。

## 解决的问题

AI 编程助手最常见的翻车场景：理解偏了直接动手、做完不验证、出错脑补答案。这套规范通过**五阶段执行流水线 + 前置三道闸门**把不确定性锁死在动手之前。

## 核心设计：坍缩生成空间

不是让 AI "更专注"，而是**约束输入分布**。开放式问题让模型在巨大可能性上铺概率，枚举成 2-4 个互斥选项后，后续生成被压进离散子空间——输出更聚焦、更准确。

## 快速概览

```
动手三问（能不能干 → 值不值得 → 有没有更省的办法）
    ↓
读 → 理解确认 → 方案选择 → 执行 → 验证
       ↑            ↑
    用户确认      用户选择
```

- **阶段一 读**：只读不猜，收集信息
- **阶段二 理解确认**：把理解写出来，用户确认对不对
- **阶段三 方案选择**：列 2-4 个互斥方案，用户选一个
- **阶段四 执行**：确认后才动手
- **阶段五 验证**：跑命令确认通过，不许说"应该可以了"

## 适用平台

规范本身平台无关。结构化确认工具按平台适配：

- **Claude Code** → 使用 `AskUserQuestion`
- **Hermes** → 使用 `clarify`
- **其他平台** → 查找对应的交互式选项工具

## 快速开始

### 方式一：作为 Claude Code / Hermes Skill

将 `agent-rules.md` 放到你的 Skills 目录，Agent 会在每次任务时加载执行规范。

### 方式二：作为 CLAUDE.md / 项目指引

将核心条款复制到项目的 `CLAUDE.md` 或 `AGENTS.md` 中。

### 方式三：手动参考

直接阅读 `agent-rules.md`，把你的个人偏好替换进去。

## 自定义

规范设计为模板，三步适配：

1. **路径替换**：把临时文件目录、项目目录、Agent 存储路径换成你的
2. **工具适配**：把结构化确认工具换成你所用平台的对应工具
3. **条款增删**：通用原则可按需增减，五阶段流水线建议保留完整

详见 [agent-rules.md](agent-rules.md) 末尾的自定义指南。
